## Key Design Decisions

### 1. Data-Driven Architecture
- **Why**: Avoid hard-coded HTML for scalability
- **Benefit**: Add new articles by editing JSON only
- **Future-proof**: Ready for 100+ articles

### 2. Pure HTML/CSS/JS
- **Why**: No framework dependencies
- **Benefit**: Faster loading, easier maintenance
- **Compatibility**: Works on all browsers

### 3. Mobile-First Responsive Design
- **Why**: Majority of users access via mobile
- **Implementation**: CSS media queries for 3â†’2â†’1 column flow
- **Touch-friendly**: Larger buttons, swipe-friendly modal

### 4. Performance Optimization
- **Lazy loading**: Load articles in batches
- **Search efficiency**: Client-side filtering
- **Modal caching**: Articles stay in memory once loaded

### Troubleshooting
- **JSON not loading**: Check server logs, verify file exists

- **Modal not opening**: Verify event listeners are attached

## Lessons Learned

### Technical
1. **JSON serving**: Need explicit route in Node.js server
2. **CORS**: Not an issue since same-origin
3. **Mobile performance**: Lazy loading essential for 100+ articles

### User Experience
1. **Search placement**: Top of section for immediate visibility
2. **Share feedback**: Visual confirmation important
3. **Modal navigation**: Multiple close methods improve UX

### Project Management
